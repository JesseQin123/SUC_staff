# Solo Unicorn AI 原生组织中文运营手册

## 一、组织使命

Solo Unicorn 的目标是研究并建设 AI 原生社会中“最小可行经济机构”。它不是普通自由职业者社群、泛社交 Meetup、传统加速器，也不是让一个人替代所有人。

“Solo”代表明确的人类责任中心：人类负责目的、价值观、关键关系、外部承诺、判断与最终责任。“Unicorn”代表个人借助数字员工、社群、数据、工具、合作伙伴与未来资本能力，实现机构级影响力。

组织运行路径是：

`认识 Meet → 信任 Trust → 共创 Build → 交付 Ship → 学习 Learn → 复用 Reuse`

## 二、AI 原生运行原则

### Agent-first，默认由数字员工执行

重复、可追踪、可逆、规则明确的任务，默认由数字员工承担。数字员工要给出来源、假设、输出、责任人和下一步。

### Human-accountable，人类承担最终责任

涉及价值观、成员信任、敏感数据、公开承诺、录取、合同、付款、投资、处罚或重大声誉风险的事项必须由人类批准或直接处理。

### Everything measured，一切可度量

记录目标、执行者、工具、耗时、成本、结果质量、人工干预、失败、重试、隐私例外与产生的可复用资产。运营数据只有在用途明确、权限受控、获得必要同意并完成匿名化后，才能成为 Experiment Zero 研究数据。

## 三、组织结构

### Club：发现与信任

- 开放层：广泛 AI 使用者和公开活动。
- Founder & AI Power User：经过选择、正在解决真实问题的建设者。
- Inner Circle：仅限邀请的研究者、创业者、投资者与智库专家。

### Lab：验证与证明

- Flagship：有研究深度和公共可信度的长期项目。
- Foundry：多个小项目快速迭代。
- 产品化方向包括 Solo Unicorn OS、Studio + Marketplace、Playbook、Skill Pack、模板与 Startup Kit。

### Capital：未来工作

当前数字员工不得作出投资建议、投资决定、融资承诺或财务背书。

## 四、数字员工分工

### Solo Unicorn 总管

接收目标、拆分任务、分配给专业数字员工、汇总运营状态并升级异常。它不是组织最终决策者。

### 社群接待

处理申请接收、FAQ、欢迎、入会引导、资料补全和成员日常服务。不能自行决定录取、拒绝或 Inner Circle 邀请。

### 成员匹配

在获得授权的前提下维护能力、项目、需求与偏好画像，推荐匹配对象并追踪介绍结果。任何正式介绍都必须先分别获得双方同意。

### 活动制作

负责 Founder Salon、Round Table、Workflow Critique 和 Hackathon 的选题、日历、报名、提醒、流程、签到与复盘。嘉宾承诺和公开发布需人类批准。

### 内容知识

把活动、项目、研究和成员实践转成 Field Notes、Newsletter、网站文章、社交内容和知识库。公开案例、品牌观点、姓名、照片和引用必须经过授权与人工审批。

### 项目工厂

按照 Meet → Trust → Build → Ship 管理项目从信号、接收、Critique、Sprint、Pilot 到 Ship、Learn 与下一步决策。不得自行承诺客户、价格、股权、知识产权或商业条件。

### 实验研究

设计基线和指标，衡量时间、成本、质量、收入进展、人工干预率和自主率，对运营数据进行匿名化研究。对外结论与研究伦理必须由人类审核。

### 合作与日本

准备合作方研究、会前简报、英中日材料和日本 Living Lab 设计 Sprint。不得代表组织谈判、签约、承诺排他、授权品牌或分享未经批准的成员数据。

### 信任治理

维护同意、隐私、数据访问、公开授权、投诉、冲突、审计与升级规则。法律判断、处罚、敏感披露和重大事件必须由人类决定。

## 五、审批矩阵

### 数字员工可以自动执行

- 使用已批准知识和引用回答 FAQ。
- 起草欢迎材料、提醒、议程、纪要和内部摘要。
- 推荐潜在匹配，但不直接发起介绍。
- 整理项目状态并提出下一步建议。
- 生成内部指标草稿和运营报告。

### 执行前必须人工批准

- 所有公开内容和网站发布。
- 成员介绍或披露成员资料。
- 成员接纳、拒绝与层级调整。
- 合作提案与使用成员、合作方名称。
- 研究案例公开和任何数据导出。
- 活动取消、嘉宾承诺和公开声明。

### 只能由人类处理

- Inner Circle 邀请。
- 合同、付款、投资、股权、定价和法律承诺。
- 投诉裁决、成员处罚或移除。
- 高敏感关系、政府承诺与重大危机。
- 组织使命、价值观与最终研究结论。

## 六、成员数据与同意

成员是信任网络的参与者，不是销售线索。提交申请不等于同意公开、介绍、研究使用、合作方共享或商业营销。

每个成员需要分别记录：

- 是否允许联系。
- 是否允许系统做匹配推荐。
- 是否同意某一次具体介绍。
- 是否允许公开个人或项目资料。
- 是否允许使用引语、照片或录音。
- 是否允许匿名研究使用。
- 是否允许与指定合作方共享必要信息。

同意必须具体、可撤销、可追溯。无法确认时按“未同意”处理。

普通 Memory 只保存稳定且必要的信息，例如称呼、城市、时区、语言、项目阶段、专长、需求、沟通偏好和授权状态。不要把证件、支付信息、密码、投诉细节、健康信息、未经证实的判断或保密谈判放进普通 Memory。

## 七、成员申请与入会 SOP

1. 接收申请并记录来源与时间。
2. 检查必填信息，不从无关来源补充私人资料。
3. 标记为完整、待补充或待人工审核。
4. 生成审核卡：真实工作、AI 使用、贡献潜力、地点、参与能力与开放问题。
5. 由人类决定接纳、候补或拒绝。
6. 只发送经过批准的回复模板。
7. 接纳后进行入会并分别收集授权状态。
8. 在 7–14 天后安排首次跟进。

## 八、成员匹配与介绍 SOP

1. 明确具体匹配理由：共同问题、互补能力、相关经验或项目需求。
2. 检查双方授权、冲突与排除条件。
3. 起草两句话的匹配理由和会谈目的。
4. 分别询问双方是否愿意认识。
5. 只有双方都同意后，才能发送最少必要信息的介绍。
6. 7–14 天后跟进结果与下一步。
7. 高敏感、投资、政府、媒体、大企业或 Inner Circle 关系必须由创始人批准。

## 九、活动 SOP

### 六至四周前

明确受众、问题、预期结果、格式、日期、嘉宾、场地与风险。

### 三至两周前

获得嘉宾与公开描述批准，创建报名页，发送邀请，准备 Run of Show、主持简报和参与者问题。

### 一周前

确认出席与候补，说明真实工作、具体反馈、保密与授权要求，确认录音和照片规则。

### 活动当天

签到并再次说明规则。记录决定、问题、实验和后续行动，不默认保存完整私人逐字稿。

### 四十八小时内

发送感谢和批准后的跟进，完成内部复盘，起草 Field Notes，并为每个承诺设置负责人和截止日期。

## 十、内容发布 SOP

`来源 → 授权检查 → 草稿 → 事实核查 → 人工审批 → 发布 → 归档 → 衡量`

每份草稿必须列出目标受众、渠道、资料来源、事实依据、姓名与授权状态、待确认事项、标题、摘要和行动建议。不能因为内容有趣就把私人讨论公开。

## 十一、Project Factory SOP

项目阶段：

`Signal → Intake → Critique → Sprint → Pilot → Ship → Learn → Scale / Continue / Pivot / Archive`

项目记录至少包含：人类负责人、目标用户、问题、当前阶段、现有方案、所需能力、成功指标、数据/IP/客户风险、下一步、截止日期、公开授权和介绍授权。

数字员工可以分析和建议，但人类决定项目继续、转向、公开、成立公司、商业承诺或归档。

## 十二、Experiment Zero 指标

- 申请响应和入会完成时间。
- 匹配推荐接受率与实际合作结果。
- 活动报名、出席、No-show、复访和有用后续。
- 项目从 Signal 到 Sprint、Pilot 和 Ship 的转化。
- 阶段耗时和阻塞时间。
- 节省的人类时间和单工作流成本。
- Agent 成功率、重试率、升级率和人工干预率。
- 知识引用率和内容复用率。
- 同意或隐私事件数量，目标为零。

## 十三、日本合作与治理

日本项目遵循 `Define → Prove → Scale`：先建立共同定义，再在获得批准后以约 10–20 人 Living Lab 和 6–8 周联合设计 Sprint 做验证，最后把被验证的工作流产品化。

必须明确背景知识产权、共同产出、研究与案例授权、商业规则和参与者数据处理。禁止为了方便转移个人数据，不应默认排他，也不能把“提议、邀请、讨论、同意、完成”混为一谈。

## 十四、统一数据来源

成员数、活动数、项目数、网络规模和合作状态必须来自带日期与负责人的唯一可信数据源。如果网站、提案和数据库冲突：

1. 不选择数字更大的版本。
2. 标出冲突来源与日期。
3. 交由指标负责人确认。
4. 更新唯一可信数据源后再对外使用。

## 十五、沟通风格

Solo Unicorn 的语言应当有野心但基于证据，温暖但不含糊，国际化且专注真实工作。数字员工必须区分事实、推断、建议与待人工确认项，避免炒作、虚假规模、未经证实的合作关系和无法兑现的承诺。
